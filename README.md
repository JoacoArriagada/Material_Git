## 🎯 Propósito y Contexto

Este material está diseñado para los estudiantes de la asignatura **ICC 531**. El objetivo es transitar de un modelo de "archivos sueltos" a un flujo de trabajo profesional.

### Un poco de historia...
Antes de la existencia de Git, el desarrollo de sistemas complejos como Linux dependía de herramientas propietarias. 
> *“Antes de Git, los desarrolladores de Linux usaban BitKeeper, un sistema cuyo uso era gratuito anteriormente, pero en 2005, su titular decidió revocar dicha gratuidad.”*

Ante esta limitación, **Linus Torvalds** creó Git: un sistema descentralizado que devolvió el control a los desarrolladores y optimizó el flujo de trabajo global.

## 🚀 Comandos de Supervivencia

Estos son los pilares que utilizaremos en nuestros laboratorios prácticos de los días martes.

| Comando | Acción |
|---|---|
| `git init` | Inicia un repositorio local. |
| `git clone [url]` | Descarga un proyecto existente. |
| `git add .` | Prepara todos tus cambios para ser guardados. |
| `git commit -m "mensaje"` | Crea un hito en la historia de tu proyecto. |
| `git push origin main` | Sube tus avances a la nube (GitHub). |
| `git pull origin main` | Trae los cambios de tus compañeros a tu local. |
