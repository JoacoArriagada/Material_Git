## 🎯 Propósito del Repositorio

Este repositorio nace como **material de apoyo** para los estudiantes de segundo año de la asignatura **ICC 159 — Introducción a la Ingeniería Informática**.

### ¿Por qué aprender control de versiones?

El control de versiones es una de las competencias fundamentales de cualquier profesional del área informática. Con **Git** podrás:

- 📁 **Organizar tu trabajo** de forma profesional y estructurada.
- 🔄 **Registrar el historial** de cada cambio realizado en tu código.
- 🤝 **Colaborar en equipo** sin sobrescribir el trabajo de otros.
- ⏪ **Revertir errores** fácilmente cuando algo sale mal.
- 🌐 **Compartir proyectos** con el mundo a través de plataformas como GitHub.

> 💡 *No importa si estás comenzando desde cero — todos los grandes proyectos empezaron con un simple `git init`.*

## 🚀 Comandos Git de Supervivencia

Estos 5 comandos son los que **más usarás** en tu día a día. ¡Apréndetelos y practica con ellos!

| # | Comando | Sintaxis básica | ¿Para qué sirve? |
|---|---------|-----------------|------------------|
| 1 | `git init` | `git init` | Crea un directorio oculto `.git` en la carpeta actual, convirtiendo el proyecto en un repositorio Git que podrá rastrear su historial de cambios. |
| 2 | `git add` | `git add <archivo>` o `git add .` | Prepara los archivos modificados para ser confirmados (los mueve al *staging area*). |
| 3 | `git commit` | `git commit -m "mensaje"` | Guarda una instantánea de los cambios preparados con un mensaje descriptivo. |
| 4 | `git push` | `git push origin <rama>` | Sube los commits locales al repositorio remoto (por ejemplo, GitHub). |
| 5 | `git pull` | `git pull origin <rama>` | Descarga y fusiona los cambios del repositorio remoto en tu rama local. |

### 🔁 Flujo de trabajo típico

```bash
# 1. Crea o accede a tu carpeta de proyecto
git init

# 2. Modifica o crea archivos, luego agrégalos
git add .

# 3. Confirma los cambios con un mensaje claro
git commit -m "feat: agrega función de bienvenida"

# 4. Sube tus cambios a GitHub
git push origin main

# 5. Descarga los cambios de tus compañeros
git pull origin main
```
